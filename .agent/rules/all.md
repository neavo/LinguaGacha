1. 原则
- 立足于第一性原理剖析问题，善用工具以提升效率
- 以事实为最高准则，若有任何谬误，恳请坦率斧正，助我精进
- KISS (简单)、YAGNI (拒过度设计)，崇尚简洁与可维护性，避免过度工程化与不必要的防御性设计

2. 流程
- 关键逻辑必写**单行注释**解释为什么
- 所有回复、注释、思考过程及任务清单**必须使用中文**
- 严格遵循**构思方案→提请审核→分步骤执行**的作业顺序，在实际执行任务前，必须完成前期调研并厘清所有疑点

3. 命名
- 优先使用枚举值，禁止使用魔术数/魔术字符串
- 强制跟随文件现有风格，无参考默认 `snake_case`
- 变量名须意图明确，函数名必含动词：
  - ❌ `data` `info`
  - ✅ `user_profile` `get_status`

4. 风格
- 严禁深层 `if` 嵌套，优先使用 Early Returns
- DRY策略：代码重复 <3 次保留以降低阅读负担，≥3 次才提取函数

5. Python
- 使用 `uv` 管理和运行 Python 项目
- **必须**：
  - 标注类型
- **优先**：
  - f-string
  - 列表推导式
  - Top-level Imports
